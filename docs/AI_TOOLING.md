# Se'kret Bip demo — MCP and GitHub Models

This repository is an interactive demonstration surface, not the production Se'kret Bip application or backend.

## MCP servers

- **GitHub:** repository, pull-request, Actions, and security evidence with lockdown mode.
- **Figma:** approved demo frames and visual context.
- **Playwright:** pinned isolated Chromium for phone-width layout, navigation, and accessibility verification.

Supabase and Cloudflare production MCP servers are intentionally absent. Production database, Worker, and release evidence belong to `jussray/Sekret-Bip`, not this demo.

## GitHub Models

GitHub Models may evaluate invented demo copy and synthetic interaction states.

- The manual workflow uses the automatic `GITHUB_TOKEN` with only `contents: read` and `models: read`.
- Local or Codespaces use may store a fine-grained `models:read` PAT as `GITHUB_MODELS_TOKEN`.
- Never commit a token.

Do not add or send real teen/parent messages, journals, voice transcripts, Bridge content, Circle content, identity data, production telemetry, safety alerts, or database exports. A demo must remain synthetic even when it convincingly imitates a production flow.

A model may review wording or interaction clarity. It may not be represented as the production companion, safety system, database, or release authority.
